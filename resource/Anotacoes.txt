Curso de Java 40: Orientação a Objetos: Herança e Polimorfismo: sobrecarga de métodos

	- Polimorfismo é quando um classe filha sobre escreve um metodo declarado na classe pai
	- uma classe filha pode ser instanciada do tipo Pai mas construida com o construtor filha
		Ex: Pessoa - classe pai
			Aluno - classe filha (extends Pessoa)
			Professor - classe filha (extends Pessoa)
			
			Pessoa pessoa = new Pessoa();
			Pessoa aluno = new Aluno();
			Pessoa professor = new Professor();
			
			pessoa.metodoSobrescrito();  	// aqui o metodo executado sera o declarado na classe Pessoa
			aluno.metodoSobrescrito();		// aqui o metodo executado sera o declarado na classe Aluno, caso esse metodo foi sobre escrito, se nao será executado o da classe Pessoa
			professor.metodoSobrescrito();	// aqui o metodo executado sera o declarado na classe Professor, caso esse metodo foi sobre escrito, se nao será executado o da classe Pessoa
			
			
Curso de Java 41: Orientação a Objetos: Herança: classes abstratas

	- Classes abstratas não podem ser instanciadas, elas devem ser extendidas e ai suas classes filha serão instanciadas com metodos e atributos da sua classe pai que é abstrata
	- Um classe abstrata pode ter um método abstrado, que nao deve ser implementado na propria classe, e devera ser implementado diretamente nas classes que extederam a classe abstrata
	
	
Curso de Java 42: Orientação a Objetos: Palavra chave final

	- Palavra chave final serve para indicar que a classe nao pode ser extendida
	- No caso de atributos significa que o atributo nao pode ser modificado, ou seja, é considerado uma constante.
	- É boa prática ter o nome das constantes de maiusculo.
	
		Ex: É comun ter uma classe só de constantes.
		
			public final class Constantes {
				
				public static final String URL_PAGINA = "www.google.com"
			
			}

Curso de Java 44: Orientação a Objetos: Interfaces
	
	- Interface : equivalente a uma classe abstrata que contém apenas métodos sem nenhum atrubuto.
	- É utilizado pois no Java não é possivel extender mais de uma classe
	- Interface define qual o comportamente uma classe precisa implentar, a interface apenas define a assinatura
	- Declaração Interface:
	- É comum ter uma interface de constantes apenas (Todo atributo dentro de uma interface é sempre public static e final).
	
	public interface ExemploInterface {
		
		void metodoExemplo();
		double outroMetodoExemplo();
		
	}
	
	- implements é a palavra chave para implementar uma Interface.
	- para implementar mais de uma interface:
	
	public class ClasseDeTeste implements Interface1, Interface2 {
		...
	}
	
	- Exemplo de interface de constantes:
	
	public interface Constantes {
	
		String CONSTANTE_01 = "bla bla bla";
		String CONSTANTE_02 = "aaaaa";
		...	
	}
	
	
Curso de Java 45: Orientação a Objetos: Casting e instaceof

	- Upcasting: É quando uma classe pai (ou acima) de uma classe filha recebe uma instancia.
		Ex:
			Aluno aluno = new Aluno();
			Object o = aluno;
		
		Upcasting automatico:
			Pessoa aluno = new Aluno();
		Upcasting Manual:
			Pessoa aluno = (Pessoa) new Aluno();
	
	- Downcasting: É quando uma classe filha recebe uma instancia de uma classe pai (ou acima)
		Ex:
			Object obj1 = "Aqui é uma String";
			String s1 = (String) obj1;
			
	- instanceof: Serve para verificar se uma objeto instaciado é de um certo tipo
		Ex:
			Pessoa p1 = new Pessoa();
			
			p1 instanceof Pessoa - deve retornar true pois p1 é uma instancia da classe Pessoa.


Curso de Java 46: Orientação a Objetos: Interface e herança

	- "Existe herança multipla no Java? Não, o Java não permite, porem é possivel implentar mais de uma interface."
	- Possivel que uma interface extenda a outras interfaces.
		Ex:
			public interface InterOne extends Interface1, Interface2 {
				...
			}
	- Uma classe pode extender apenas uma classe porém pode implementar quantas interfaces forem necessárias
	- Uma interface pode extender quantas interfaces for necessarias, a primeira classe que implementar essa interface deverá implementar todos os metodos herdados pela primeira interface.
	- Interfaces não tem construtores.


Curso de Java 47: Exceptions: try, catch

	- códigos dentro do bloco 'try' são codigo que podem dar erros
	- o 'catch' serve para tratar o erro que pode ter acontecido
	- Nao tratar erros fazer os programas terminarem


Curso de Java 48: Exceptions: multiplos catch

	- É possivel usar mais de um 'catch' após um 'try'
	- O primeiro 'catch' tem uma prioridade maior, ou seja, deve ser um erro mais especifico
	- Por último deve ficar os erros mais genéricos


Curso de Java 49: Exceptions: finally

	- o bloco 'finally' é sempre executado após o 'try' ou 'catch'
	- caso exista um código similar a 'System.exit(0)' dentro dos blocos 'try' ou 'catch' o 'finally' não será executado
	
	
Curso de Java 50: Exceptions: stacktrace e throws

	- Execption é a classe de erro utilizada para pegar todos tipos de erro, como sendo genérica
	- e.getMessage() é o método para buscar a causa do erro
	- e.printStackTrace() é o metodo que é utilizado para saber onde o erro aconteceu
	- Error: resulta na finalização do programa
	- Exception: possível tratar o erro e continuar execução do programa
